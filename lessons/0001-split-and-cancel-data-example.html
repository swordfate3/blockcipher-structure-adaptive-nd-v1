用 4 条明文看懂 Split-and-Cancel

目标不是先记住“ANF、单项式预测、左核”这些名字，而是亲手看见：单个输出不平衡时，几个输出组合起来仍可能严格平衡。

学完这一页，你应该能区分三个数：一条密文内部的掩码奇偶值、一个输入集合上的积分 XOR、同一结构跨多把密钥的经验平衡率。

第一层：一条密文的 4-bit 输出变成 1 bit

先固定输出 mask 为 1111。对一条 4-bit 输出 c3c2c1c0，计算：

g(c) = c3 XOR c2 XOR c1 XOR c0

这一步只把一条密文压成一个 bit，还不是积分。

输入输出 nibble逐 bit 计算g(c)
0010011 XOR 0 XOR 0 XOR 10
0100110 XOR 0 XOR 1 XOR 10
1001000 XOR 1 XOR 0 XOR 01
1100010 XOR 0 XOR 0 XOR 11

第二层：在同一个输入结构内部，对全部明文再 XOR

现在把 00, 01, 10, 11 看成一个活动 2-bit 输入结构。对四条明文的 g(c) 再做一次 XOR：

IS(g) = 0 XOR 0 XOR 1 XOR 1 = 0

这时才能说：在这把密钥、这个输入结构和这个输出函数下，观测到零和。

积分语境里的“平衡/zero-sum”指 XOR 结果为 0，也就是 1 的数量为偶数，不一定要求 0 和 1 各占一半。比如 0000 和 1111 的 XOR 都是 0。

第三层：为什么有限密钥实验不是数学证明

换一把密钥，四条输出也会变化。假设对同一个结构测试 8 把密钥，得到：

密钥K0K1K2K3K4K5K6K7
积分结果 q(K,S,g)00000011

这里：

经验平衡率 P(q=0) = 6 / 8 = 75%
经验非平衡率 P(q=1) = 2 / 8 = 25%

这正接近我们当前 Innovation 2 网络学的东西：根据结构字段，预测它在新密钥上更可能排在高平衡率候选中。

但它不能推出：

对所有密钥 K，都有 q(K,S,g) = 0

论文的目标比 75% 更严格：它要用符号方法证明“所有允许的密钥都为 0”，或者明确写出哪些密钥条件保证为 0。

第四层：先用真实输出表看“组合抵消”

先忘掉密码内部公式，只看两个输出 bit c0 和 c1：

输入c0c1h = c0 XOR c1
00000
01101
10011
11000
跨集合 XOR110

c0 单独不平衡，c1 单独也不平衡，但是：

IS(c0 XOR c1) = IS(c0) XOR IS(c1) = 1 XOR 1 = 0

这就是 Split-and-Cancel 最朴素的现象：两个坏结果可以因为相同残留出现两次而抵消。

第五层：论文为什么需要“未决项矩阵”

真实密码不可能把所有明文、所有密钥全部枚举。论文把最后一轮精确展开。它的玩具例子是：

y0 = x0 XOR k0 XOR x1*x2 XOR x2*x3
y1 = x1 XOR k1 XOR x1*x2 XOR x1*x3
y2 = x2 XOR k2 XOR x2*x3 XOR x1*x3

前 p 轮的 oracle 已经证明 x0,x1,x2,k0,k1,k2 的积分 XOR 都是 0。剩下三个暂时无法证明为 0 的项，记成：

A = I_S(x1*x2)
B = I_S(x2*x3)
C = I_S(x1*x3)

A/B/C 不是默认等于 1。它们的意思只是“目前不知道是 0 还是 1”。于是三行的积分结果可以写成：

I_S(y0) = A XOR B
I_S(y1) = A XOR C
I_S(y2) = B XOR C

现在三行一起 XOR：

(A XOR B) XOR (A XOR C) XOR (B XOR C) = A XOR A XOR B XOR B XOR C XOR C = 0

关键在于：不需要知道 A/B/C 的真实值。每个未知项都出现两次，所以必然抵消。

动手切换 A、B、C

任选三个未知积分值。观察前三项会改变，但最后的组合始终是 0。

IS(y0)0
IS(y1)0
IS(y2)0
三行组合0

第六层：矩阵只是把上面的式子压缩记录

按列记录每行包含哪些未决项：

输出量A = x1*x2B = x2*x3C = x1*x3
y0110
y1101
y2011

选择向量 lambda = [1,1,1] 表示三个输出量都选。三行按位 XOR：

lambda M = [1,1,1] M = [0,0,0]

所以 lambda 在矩阵的左核里，而它对应的输出组合就是：

G = y0 XOR y1 XOR y2

“求左核”并没有创造新的密码学事实；它只是一次性找出所有能让每一列出现偶数次的行组合。

第七层：为什么会得到 1/4 弱密钥

加入最后一轮密钥后，某个组合可能没有完全消掉，而是剩下：

IS(G) = k1*A XOR k0*B

A/B 仍然未知。逐个看两个密钥 bit：

k0k1剩余表达式是否保证为 0
000*A XOR 0*B = 0保证
01A未知
10B未知
11A XOR B未知

四种密钥取值中，只有 k0=k1=0 可以无条件保证，所以覆盖比例是：

1 / 4 = 2^-2

这就是论文里“弱密钥 portion 至少为 2^-2”的直观来源。它不是训练准确率 25%，而是满足精确密钥条件的密钥子空间占全部密钥空间的四分之一。

一分钟自测

问题 1：8 把密钥中有 6 把得到 q=0，能否称为确定性积分？

不能。只能报告有限样本下的经验平衡率为 75%。确定性积分要求对声明范围内的所有密钥成立，或者有符号证明。

问题 2：矩阵里一个 1 是否表示该单项式的积分值一定为 1？

不是。矩阵里的 1 表示“这个输出表达式包含该未决单项式”。未决意味着 oracle 暂时不能证明其积分和为 0，并不等于真实值是 1。

问题 3：为什么 lambda=[1,1,1] 能给出证明？

因为三行相加后，每个未决列都出现两次，F2 中 1 XOR 1 = 0。所有未决项消失，其余项此前已经由 prefix oracle 证明为零和。
