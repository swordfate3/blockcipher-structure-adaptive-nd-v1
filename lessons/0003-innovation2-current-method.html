创新2当前到底如何做

从一批结构化明文出发，严格判断哪些输出bit保持XOR平衡，再让一个读取前三轮代数特征和真实SPN拓扑的小网络一次预测64个输出bit的平衡谱。

当前已确认的方法叫前缀引导逐节点平衡谱算子（Prefix-Guided Nodewise Balance-Profile Operator，PG-NBPO）。它在PRESENT-80四轮和GIFT-64四轮分别训练、双seed通过；它不是七到九轮攻击。

1. 它预测什么，不预测什么

创新2不预测某条明文的具体密文，也不做“真结构对随机数据”的二分类。一个查询是：

给定：密码、轮数r、8个活动输入bit组成的cube S、输出bit j
询问：对S内部全部2^8=256条明文加密后，输出bit j的XOR是否恒为0？
B_r(S,j)=XORx in S bit_j(E_K^r(x))

网络对同一个结构一次输出64个logit，对应：

[B_r(S,0), B_r(S,1), ..., B_r(S,63)]
这里的“平衡”是一个结构内部全部明文输出的XOR为0。不是四个输出bit先异或成一个bit，也不是不同结构之间互相XOR。

2. 一条真实PRESENT数据如何生成

项目中的真实训练结构cube_026有8个活动bit：

active_bits = [4, 7, 28, 38, 40, 44, 53, 59]
active_mask = 0x0820114010000090

固定其余56个bit为一个inactive offset。令8个活动变量为a0...a7，每个取0或1：

P(a)=offset XOR (a0<<4) XOR (a1<<7) XOR ... XOR (a7<<59)

遍历所有00000000...11111111，得到256条明文。下面使用标签库中的真实反例密钥和offset：

XOR all 256 output bit 12 = 1

所以这个具体密钥和offset已经证明bit 12不是“对所有情况恒平衡”。bit 33在这里为0，但仅凭一次为0还不能证明它对所有密钥和offset都平衡。

3. 标签为什么有三种

存储值含义判定依据是否训练
1 / positive严格平衡完整8变量cube单项式不可能出现在目标输出ANF支撑中是
0 / negative不是全称平衡找到真实PRESENT-80调度密钥和inactive offset，使256条输出XOR为1是
-1 / unknown尚未判定既没有全称正证书，也没有具体反例否

cube_026, bit 12的标签是0，证书是concrete_key_offset_masked_xor_one。cube_026, bit 33的标签是1，证书是full_cube_monomial_absent_from_support_overapprox。

正类要求“所有密钥、所有inactive offset都成立”；负类只需一个可复算反例。unknown绝不偷改成negative。

4. 神经网络看不到最终答案

目标是四轮平衡标签，但网络只看第三轮结束时的代数前缀。对每个输出bit节点，追踪8个活动变量经过前三轮S-box和P-layer后可能形成的ANF单项式支撑。

cube_026, bit 33的真实13维第三轮特征为：

[0.375, 0.375, 0.375, 0.375,
 0.0104167, 0.0833333, 0.2395833, 0.3333333,
 0.2395833, 0.0833333, 0.0104167, 0, 0]

前4项是支撑规模统计：mean、max、normalized sum、union size。这里都是0.375，表示该单位输出节点的支撑中有0.375 x 256 = 96种活动变量单项式。

后9项是次数0到8的归一化直方图。把它乘回96，可读成：

degree 0..8 monomial counts = [1, 8, 23, 32, 23, 8, 1, 0, 0]
第4轮最终证书、具体witness、密钥、offset、256条输出XOR和标签本身都不进入特征。否则网络只是在读取答案。

5. 4,795参数的PG-NBPO如何计算

输入              batch x 64 nodes x 13 features
LayerNorm         每个节点归一化13维特征
Linear            13 -> 32，得到节点状态h_j
消息传递          重复2次
输出头            LayerNorm(32) -> Linear(32,1)
输出              batch x 64 logits

每次消息传递给输出节点j准备三份32维信息：

  1. 自身状态：h_j。
  2. S-box/nibble上下文：和j同属一个4-bit nibble的4个节点状态均值。
  3. P-layer前驱：真实P置换中流向j的前驱节点状态。

三份信息拼成96维，经过共享更新块：

Linear(96,32) -> GELU -> Dropout(0.10) -> Linear(32,32) -> residual -> LayerNorm

64个节点共享同一套参数，所以模型学习的是“哪类代数支撑沿SPN结构传播后容易保持平衡”，不是为每个bit死记一套分类器。

6. 为什么同时训练三行模型

模型行参数量关系输入要排除的解释
independent4,795不用真实nibble/P关系单节点13维特征已经足够
true4,795真实S-box分组和P-layer候选方法
corrupted4,795同family的确定性错误P-layer任意消息混合都能提升

hidden、message steps、dropout、优化器、epoch、seed和参数量全部相同。只有关系图不同，因此true-corrupted才是“真实密码拓扑是否有独立价值”的关键差值。

7. 数据矩阵和损失如何组织

一个batch元素是一整个结构，不是单条(structure, output bit)边：

prefix_features : structures x 64 x 13
profile_targets : structures x 64，观测位置为0/1，其余为-1
profile_observed: structures x 64，严格正负标签位置为true

网络产生64个logit，但损失只看profile_observed=true的位置：

L = sum(M_sj * BCE(logit_sj, target_sj)) / sum(M_sj)

checkerboard同时保证：训练和验证结构互斥；每个结构和每个输出bit的正负数量配平；重复边为0；global、output-only和active-bit边际AUC接近0.5。

9. 最新九轮E98和主方法是什么关系

E98没有训练PG-NBPO。它审查的是另一条高轮桥接假设：能否把公开ATM的PRESENT轮函数九轮广义积分关系做成正例-未标注候选排序。

ATM序列化关系       = 470
GF(2)并集秩         = 468
严格文件留出正例    = 24
有正例的留出组      = 3 / 8
达到每组8条的组     = 1 / 8
E99神经排序          = closed

候选构造和捷径控制通过，但独立正例太少，而且470与468的秩差异未解释。因此现在不能把九轮关系重复增强后训练，也不能把未标注候选叫作负例。

当前最强创新2结果仍是PRESENT/GIFT四轮PG-NBPO。九轮E98是“高轮数据是否够资格训练”的审计，不是九轮神经预测成绩。

10. 创新点到底落在哪里

  1. 任务创新：从预测具体密文或真假样本，改成一次预测64维积分输出平衡谱。
  2. 标签创新：全称正证书、具体负见证、unknown保留，不用有限密钥投票冒充证明。
  3. 表示创新：只输入目标轮前的ANF支撑规模和次数谱，避免最终标签泄漏。
  4. 网络创新：把64个输出bit作为节点，沿真实S-box分组和P-layer共享消息传递。
  5. 评测创新：structure-disjoint checkerboard、强边际基线和同参数错误拓扑控制。
  6. 证据边界：双密码双seed确认四轮方法，同时明确报告五到九轮严格标签尚未就绪。

一分钟自测

网络的一个样本是一条明文吗？

不是。一个样本是一个8-bit活动cube结构，内部有256条明文；网络一次输出该结构的64个输出bit预测。

某个密钥下256条输出XOR为0，就能标成positive吗？

不能。positive需要全称证书；一次或有限次为0只能说明尚未看到反例。

true模型AUC高，如何证明不是单节点特征或任意图混合？

同预算比较independent和corrupted两行；两颗seed都要求true分别领先至少0.03。

E98是否已经做到PRESENT九轮预测？

没有。E98只做九轮正例-未标注数据就绪审计，因独立正例不足和470/468秩差异而暂缓。
