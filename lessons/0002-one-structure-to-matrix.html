先看懂“一个结构”，再看三行矩阵

这一课只解决两个问题：一个输入结构到底装了什么；论文矩阵的行和列分别代表什么。暂时不需要记“左核”。

术语纠正：不要说“跨结构 XOR”。准确说法是“在一个结构内部，对它包含的全部明文输出求 XOR”。多个结构之间通常不互相 XOR。

1. 一个输入结构就是一批按规则生成的明文

先用只有 4 bit 的玩具明文。规定：

前两 bit 固定为 10
后两 bit 活动，遍历 00、01、10、11
这四条明文合在一起，才叫一个结构 S1
1000
1001
1010
1011

所以：

S1 = {1000, 1001, 1010, 1011}

S1 不是一条明文，也不是四个结构。它是一个结构，内部有四条明文。

2. 在一个结构内部求 XOR

固定一把密钥 K0 和一个输出公式 g。四条明文各自加密后，g 给出一个 bit：

S1 内的明文加密后计算 g
10000
10011
10101
10110

只对 S1 盒子里的四个结果求 XOR：

q(K0,S1,g) = 0 XOR 1 XOR 1 XOR 0 = 0

这表示 g 在结构 S1、密钥 K0 下得到零和。

3. 什么才叫另一个结构

如果把固定前缀从 10 改成 11，就得到另一个结构：

S2 是另一个独立的盒子
1100
1101
1110
1111

正确做法是分别计算：

q(K0,S1,g) = XOR S1 内四条明文的结果
q(K0,S2,g) = XOR S2 内四条明文的结果
通常不会再计算 q(S1) XOR q(S2)。在当前神经数据集中，S1 和 S2 是两个候选样本。

4. 换成 PRESENT 是同一件事

PRESENT 有 64-bit，也就是 16 个 nibble。假设最后一个 nibble 活动，前 15 个 nibble 固定：

S = {
  123456789ABCDEF0,
  123456789ABCDEF1,
  ...,
  123456789ABCDEFF
}

这里有 16 条明文，但仍然只有一个结构。活动 nibble 改位置，或者固定的 15 个 nibble 改值，才会得到另一个结构。

5. 论文的三行不是三个结构

现在固定同一个结构 S，不再换盒子。论文准备检查三个不同的输出公式：

y0 = 第一个输出公式
y1 = 第二个输出公式
y2 = 第三个输出公式

三者都在同一个 S 上计算：

I_S(y0) = 把 S 内所有明文的 y0 结果 XOR
I_S(y1) = 把 S 内所有明文的 y1 结果 XOR
I_S(y2) = 把 S 内所有明文的 y2 结果 XOR
矩阵的三行是三个输出公式，不是三条明文、三把密钥或三个输入结构。

6. 把三行矩阵看成“未决项出勤表”

经过论文的前缀检查和最后一轮展开后，已证明为 0 的部分都被删掉，只剩三个暂时不知道是 0 还是 1 的残留项：

A、B、C = 三个尚未解决的积分残留

三个输出公式分别带着：

y0 带着 A、B
y1 带着 A、C
y2 带着 B、C

把“出现”写成 1，“没出现”写成 0，就是论文矩阵：

是否选择输出公式ABC
y0110
y1101
y2011
所选行 XOR 后每列剩余000
还没有选择输出公式。

请依次试三种选择：

  1. 只选 y0：A、B 各出现一次，仍然未知。
  2. 选择 y0、y1：A 出现两次抵消，但 B、C 各剩一次。
  3. 三个全选：A、B、C 都出现两次，全部抵消。
(A XOR B) XOR (A XOR C) XOR (B XOR C) = 0

所以论文得到：

I_S(y0 XOR y1 XOR y2) = 0
这不表示每条明文都满足 y0 XOR y1 XOR y2 = 0。它只表示：在结构 S 内，把所有明文的这个组合结果再 XOR，最终为 0。

7. “左核”现在可以翻译成人话

左核只是计算机用来自动回答下面这个问题的工具：

选择哪些行，能让每个未决列都出现偶数次？

本例答案是选择 y0、y1、y2。把“选/不选”写成：

lambda = [1, 1, 1]

这里的三个 1 只表示三行都选。计算机写成 lambda M = 0，人话就是“A、B、C 三列全部配成偶数对”。

一分钟自测

S1 内有四条明文，所以 S1 是四个结构吗？

不是。S1 是一个结构，内部包含四条按同一活动/固定规则生成的明文。

矩阵的 y0、y1、y2 三行是什么？

它们是同一个输入结构上检查的三个输出公式，不是三个结构或三条明文。

矩阵中 A 列的 1 表示 A 的真实值为 1 吗？

不是。1 只表示该输出公式包含残留项 A；A 的真实积分值仍然未知。
