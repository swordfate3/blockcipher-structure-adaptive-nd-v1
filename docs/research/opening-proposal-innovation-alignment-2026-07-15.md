# 开题报告创新点与当前项目对齐说明

**依据：** 《分组密码智能化检测系统设计与分析》开题报告（2025-12-23 版本）

**目的：** 固定论文要证明的贡献边界，避免将单一密码、单个网络或单次 AUC 实验误写成整篇论文的创新。

## 一句话结论

本论文不是“为 SPN 设计一个网络并把 AUC 做高”。它要交付一个三层闭环：

```text
结构类别 -> 为该类别选择/构造合适的神经区分器
          -> 用轮间密文预测补足另一种安全评估维度
          -> 由统一平台自动生成样本、训练、比较、解释和出报告
```

其中，第一层是当前 Innovation 1 的归属；第二层和第三层是独立的论文创新，不能被第一层实验替代。

## 报告原意中的三项创新

### 1. 密码结构到神经网络的匹配策略

开题报告第 1、11--12、17 页提出的不是“一种通吃的网络”，而是建立：

```text
密码的结构特征 -> 网络架构偏好 -> 适配的神经区分器配置
```

报告明确覆盖三种结构：

| 密码结构 | 报告强调的结构信息 | 要比较的网络能力 |
| --- | --- | --- |
| ARX | 模加、循环移位、异或及其累积关系 | ResNet 的残差传播、序列模型的依赖建模等 |
| SPN | S 盒非线性层与置换/线性扩散层的分离 | 局部单元、跨位置扩散和多 pair 证据的表示能力 |
| Feistel | 左右分支交替与对称迭代 | 序列/状态交替关系以及全局依赖建模能力 |

这里的“统一检测入口”应理解为统一的实验协议、数据适配器、模型注册和比较报告，而不是把三种密码强行塞进同一个固定网络。创新成立所需的证据是：在冻结的数据定义、负样本、轮数和计算预算下，给出可复现的跨结构比较，最终沉淀为可执行的选择规则。仅某个 SPN 密码上某个候选 AUC 较高，只是这条映射的一个数据点。

#### DDT 在这里是什么，不是什么

开题报告第 15 页把 DDT、差分活性追踪和高概率路径搜索列在“密码差分分析”关键技术中。它们可以用来选择输入差分、构造正样本、生成辅助先验或解释网络学到的信号，但报告没有要求把 DDT 数值表作为所有网络的固定输入。因此：

```text
DDT / 路径搜索 = 可选的数据与先验工具
结构化张量/网络 = Innovation 1 的候选实现手段
结构到架构的可复现选择规则 = Innovation 1 的最终贡献
```

项目已经停止的 dense DDT 输入路线不需要因为重读开题而恢复。当前基于原始密文对、SPN cell/P-layer 坐标或 Feistel 左右分支组织数据的路线仍然符合开题目标，只要它能在同协议基线和 shuffled/错误结构控制下证明结构归因。

#### 代表密码和最终交付密码

报告正文以 `ARX / SPN / Feistel` 定义研究范围，科学问题处以 DES 说明 Feistel 缺口，年度计划和最终创新清单又点名 AES、SPECK、SM4。合理的统一口径是：

| 层级 | 密码 | 用途 |
| --- | --- | --- |
| 机制研究/快速归因 | PRESENT、GIFT-64、DES | 结构清晰、文献和代码基线较明确，适合验证具体结构假设 |
| 开题明确的最终覆盖 | AES、SPECK、SM4 | 用于证明方法不是只在轻量级代理密码上成立，并支撑平台验收 |

DES、PRESENT、GIFT 的结果可以成为 Innovation 1 的有效证据单元，但不能代替最终对 AES、SPECK、SM4 的覆盖。尤其不能从一个 DES 结果外推整个 Feistel 类，也不能从一个 GIFT 结果外推整个 SPN 类。

### 2. 轮间神经网络密文输出预测攻击

开题报告第 1、4、12、17 页将它单列为第二项创新。它和神经区分器不是同一任务：

| 项目 | 神经区分器 | 输出预测攻击 |
| --- | --- | --- |
| 输入/目标 | 判断一组构造样本是真实差分还是随机负样本 | 给定状态，预测下一轮状态或密文输出比特 |
| 核心指标 | AUC、准确率、校准、区分优势 | 逐比特预测精度、收敛、达到随机猜测的临界轮数 |
| 报告要求的改进 | 结构适配的分类网络 | 从“明文 -> 最终密文”改为“中间轮状态 -> 下一轮状态”的迭代预测 |
| 要回答的安全问题 | 还存在可被区分的统计偏差吗？ | 扩散在哪一轮才足以让下一轮状态不可预测？ |

因此，第二项创新的最小可验证对象应是一个可控制轮数的数据生成器、一组 `state_r -> state_(r+1)` 预测任务，以及 ResNet、LSTM、Transformer 等在相同密码和轮数上的临界轮数曲线。当前差分神经区分器训练没有实现或验证这条主线。

这里还必须冻结威胁模型：网络是否得到轮密钥、训练与测试是否跨密钥、预测的是完整状态还是单比特、随机猜测基线如何计算。若下一轮状态本来就是当前状态和已知轮密钥的确定性函数，模型只是在拟合公开轮函数，不能直接写成密码安全攻击。Innovation 2 必须证明它在明确的未知信息和跨样本条件下测到了超出确定性/多数类基线的可泛化预测优势。

### 3. 一体化智能检测平台

报告第 2、11--12、17 页要求的是工程闭环：

```text
任务配置
  -> 样本生成/缓存
  -> 选择并训练或加载模型
  -> 多维评估和跨模型比较
  -> 实时进度、曲线、临界轮数图和正式报告
```

当前仓库的配置、可复现实验矩阵、JSONL/CSV、SVG、验证器、结果索引与远程监控已经构成后端实验管线的雏形；但它们不等于报告所述的“模块化前后端协同 Web 平台”。要把第三项写成论文贡献，仍需把这些能力收敛为统一任务 API/模型注册/结果数据模型，并提供可视化任务配置、进度和报告界面。

## 当前实验的准确位置

正在运行的 `i1_gift64_mainstream_performance_1m_seed6/seed7` 是创新点 1 的一个关键 SPN 证据，不是整个开题工作：

```text
对象       = GIFT-64，6 轮，SPN
数据       = 每 seed 训练 1,000,000/class、验证 500,000/class；4 个密文对/样本
输入       = 512 个原始密文对比特
负样本     = encrypted random plaintexts（严格负样本）
模型       = typed SPN scratch、两份 PRESENT -> GIFT typed transfer、
             Sun-style BiLSTM、Gohr-style ResNet
评估       = 10 epochs，恢复最佳验证 AUC，5 份全新 final-test
问题       = 在完全相同的数据和预算下，结构化 typed SPN 方法能否与主流 LSTM/ResNet 竞争？
```

这次实验比此前只和 shuffled topology 控制比较的 E4--E6 更接近“结构--架构偏好”的核心命题，因为它首次在同协议下加入主流架构基线。但它仍有明确边界：

- 一个 GIFT SPN 任务不能推出“SPN 都偏好该网络”，更不能推出 ARX、Feistel 的匹配规则。
- 它的目标指标是 AUC；外部论文若报 accuracy 且任务、样本结构不同，不能直接相减比较。
- 它没有实施轮间输出预测攻击，所以不覆盖创新点 2。
- 它产生平台可消费的实验产物，但尚未交付报告承诺的完整可视化前后端平台，所以不覆盖创新点 3。
- 训练尚在运行；在五次新鲜测试、双 seed、计划一致性验证和结果归档完成前，没有性能结论。

Feistel/DES 分支已经提供了另一类结构的本地归因证据，但尚未达到扩大规模的门槛：

```text
DES-r5 官方布局校准，2048/class：
  seed0 fresh AUC = 0.968407075
  seed1 fresh AUC = 0.964183966
  结论 = 数据、训练器和官方布局在可学习轮数上工作正常

DES-r6 官方骨干归因，2048/class：
  seed0 true/shuffled/raw = 0.524183512 / 0.512380679 / 0.500323693
  seed1 true/shuffled/raw = 0.527223508 / 0.508161743 / 0.502251625
  true-shuffled margin    = +0.011802832 / +0.019061764
  结论 = 正确左右分支映射的归因跨 seed 成立，但绝对 AUC 未达到 0.55
```

这组结果能写成“Feistel 分支语义在本地诊断中优于等容量乱序控制”，不能写成“已找到 Feistel 最优网络”，也不支持 DES-r6 远程扩样。后续 DES-r5 强信号归因进一步得到：正确八通道分支模型在两 seed 都优于等容量 shuffled，但 seed1 比官方四通道 raw 低 `0.01607` AUC。因此结构映射信号成立，额外 XOR 通道作为架构改进被拒绝。下一项更干净的问题是比较完全等容量的官方 raw 正确映射与 raw shuffled 映射，归因官方布局本身，而不是继续增加通道。

ARX/SPECK 在仓库中已有较大规模结果和 Gohr-style 基线，但目前还没有被整理成与 SPN、Feistel 完全同口径的“架构偏好单元”。旧结果可以作为起点，不能直接和不同 pair 数、差分、轮数、样本量的 GIFT/DES AUC 拼成一张结论表。

## Innovation 1 的完整证明链

Innovation 1 不是只看候选模型的绝对 AUC。每个结构至少需要同时回答四个问题：

| 证据层 | 必要比较 | 回答的问题 |
| --- | --- | --- |
| 可学习性 | 文献/官方布局或同协议强基线 | 数据和任务是否真的包含可学习信号？ |
| 结构归因 | 正确结构 vs 等容量 shuffled/错误结构 | 提升来自结构语义，还是参数量和随机波动？ |
| 架构竞争力 | 结构候选 vs ResNet、LSTM、DBitNet/AutoND 等 | 结构方法是否比主流网络更合适？ |
| 泛化与规则 | 多 seed、多轮数、至少两个相关密码或标准密码验证 | 能否从单个结果抽象成可复现的选择规则？ |

最后的论文产物不应只是一张“最高 AUC 排名表”，而应是一套可以执行的规则，例如：输入某密码的轮函数与扩散结构，系统如何组织张量、优先选择哪些网络族、保留哪些控制、在什么证据下回退到通用基线。

## 论文层面的正确推进顺序

当前 GIFT 大规模实验完成并通过归档后，先根据它的同协议结果决定 Innovation 1 的 SPN 结论强度，再按下面的论文级路线推进：

1. **完成创新点 1 的可比较证据。** 先等待 GIFT-64 双 seed 主流比较形成 SPN 性能结论；Feistel 下一步做 DES-r5 官方 raw 正确映射与等容量 raw shuffled 的最小归因，不扩大 DES-r6，也不调优已拒绝的额外 XOR 通道；ARX 则审计并重组 SPECK 的同协议强基线。之后再把已通过的结构规则落到开题点名的 AES、SPECK、SM4。每一类至少记录 AUC/accuracy、参数量、训练时间、样本总量/每类量和跨 seed 稳定性。
2. **独立启动创新点 2。** 先做小规模、可验证的 `state_r -> state_(r+1)` 基准和确定性基线，测量逐轮预测精度何时回到随机水平；再扩展到不同结构。它不能用区分器 AUC 替代。
3. **产品化创新点 3。** 将已验证的实验管线抽象为统一的任务和结果 schema，再增加模型/算法选择、任务提交、进度、曲线和报告界面。平台应消费前两项的真实结果，不应为了界面另造一套不可比的实验逻辑。

## 可写与不可写的结论

当前可写：

```text
项目正在对 SPN 的结构化表示/迁移方法进行受控验证；
E4--E6 已给出正确 topology 相对 shuffled controls 的归因证据；
GIFT-64 的百万/类同协议主流网络比较正在运行；
DES-r5 官方布局已完成强信号校准；
DES-r6 正确 Feistel 分支映射跨 seed 优于 shuffled/raw，但绝对信号不足，未获准扩样；
DES-r5 正确八通道映射跨 seed 优于等容量 shuffled，但 seed1 落后官方 raw，额外 XOR 通道已拒绝。
```

当前不可写：

```text
已经找到三类分组密码的最优网络；
typed SPN 已优于主流方法或达到 SOTA；
已经完成神经网络输出预测攻击；
已经完成开题报告所述的一体化 Web 检测平台。
```
